tipo de dado que o frontend está enviando:

{
  "items": [
    { "name": "nome.do.produto", "price": 10.00, "quantity": 3 }
  ],
  "currency": "brl"
}

o ideal é que o front envie o nome do produto e o back pegue o preço no banco do projeto

o código de backend recebe um carrinho do front, cria um sessão de pagamento no stripe e atualiza o banco de dados conforme o pagamento

para funcionar de verdade, é preciso gerar uma senha para o webhook no site oficial no stripe

o cors (app.use(cors());) foi usado para testar a api e nao deve ser implementado dessa forma em um projeto real. deve aceitar só requisiçoes oficiais

precisa juntar a estrutura do banco de dados do pagamento com os produtos e os usuarios

é preciso pensar ainda na autenticação, validação e proteção contra manipulação de dados

se for o caso de usar um catalogo de produtos fixo, pode cadastrar os produtos no site do stripe e puxar o id e preço deles no back. o site do stripe tem tutoriais sobre isso
